# 30-60-90 day rollout

## Day 0-30: Stabilize

### Objectives

- Stop review quality degradation
- Restore predictable queue flow
- Prevent high-risk merges without sufficient evidence

### Actions

1. Introduce PR evidence contract (`Intent/Risk/Evidence/Rollback`)
2. Enforce PR sizing and decomposition guidance
3. Classify PRs by review tier (T1/T2/T3)
4. Add/validate deterministic CI gates (tests + security scans)
5. Enable first-pass AI review bot with no merge permissions

### KPIs

- Time to first meaningful review
- Median PR size
- % PRs compliant with evidence contract

---

## Day 31-60: Harden

### Objectives

- Improve review depth consistency across reviewers
- Reduce concentration risk

### Actions

1. Introduce reviewer rotation for key subsystems
2. Add domain-owner sign-off for T3 PRs
3. Create checklist-driven review templates by risk class
4. Run weekly defect/revert retros on merged PRs

### KPIs

- Review concentration ratio (top 1-2 reviewers)
- T3 compliance rate (all required checks present)
- Rework churn trend

---

## Day 61-90: Institutionalize

### Objectives

- Make the model durable beyond the temporary capacity shock
- Build knowledge redundancy

### Actions

1. Codify policies in `CONTRIBUTING` and team playbooks
2. Expand subsystem ownership and backup reviewer coverage
3. Publish monthly review health dashboard
4. Tune thresholds based on actual defect and latency outcomes

### KPIs

- Post-merge regression rate vs baseline
- Queue SLA attainment rate
- Percentage of repos using standardized review model

---

## Stop conditions and rollback

Temporarily tighten merge policy (or pause non-critical changes) if:

- Regression rate exceeds agreed threshold vs baseline
- T3 PRs bypass mandatory evidence or sign-off
- Queue health collapses beyond agreed latency window

Re-enable normal flow after two consecutive stable reporting periods.
